## 1. 自動化・バッチ処理

### 夜間 Claude Code 自走パイプライン（GMOペパボ）

| 項目 | 内容 |
|------|------|
| トリガー | cron（深夜バッチ） |
| 処理内容 | PR/Issueのトリアージ・レビュー・タスク生成 |
| データストア | SQLite（書き込みは管理者のみ） |
| 出力UI | Python `http.server` による軽量カンバンボード |
| 人間の関与 | 翌朝カンバンを確認し優先度・状態を変更 |

**設計のポイント**
- 書き込みは cron → DB の一方向に限定（不正タスク混入防止）
- 人間が操作できるのは「待機」「優先度変更」などの状態遷移のみ（狭い APIエンドポイント）
- UIは複雑化を避け、チャンネルバッジ・タブ分割・マージボタンなど段階的に拡張

### GitHub Actions × Claude Code Action

- Google Apps Script が深夜に Issue を自動作成
- Issue トリガーで Claude Code Action が起動 → 自動でコードを改善して PR 作成
- 翌朝エンジニアがレビューするだけのワークフローが完成

### Claude Code Routines（公式機能）

- スケジュール・API呼び出し・GitHub イベントで定期実行が可能
- Anthropic マネージドインフラ上で動作（自前サーバー不要）

## 2. 自己改善・スキル進化

### Hermes Agent の自己評価ループ

```
タスク実行
  ↓
自己評価チェックポイント（15ツールコールごと / 複雑タスク後）
  ↓
「保存価値あり」と判定
  ↓
スキルファイル（手続き知識）を生成・更新
  ↓
次回タスクで再利用 → 40%高速化
```

**評価タイミング**
- 15ツールコールごとに起動
- 5回以上のツールコールを要した複雑タスク完了後にも起動
- スキル生成は完全自動・サイレント
- **スキル進化（既存スキルの更新）は人間レビューが必要**

### DSPy + GEPA によるスキル最適化

- GEPA（Genetic-Pareto Prompt Evolution）= 遺伝的アルゴリズム + パレート最適化
- 実行トレースから「なぜ失敗したか」「なぜ余分なステップが必要だったか」を自然言語で分析
- 候補バリアントを生成 → 評価ラウンド → パレートアルゴリズムで勝者を選択
- ICLR 2026 oral 採択技術（GEPA）

## 3. 記憶・知識管理（Three-Layer Memory）

| レイヤー | 内容 | 永続性 |
|----------|------|--------|
| セッションコンテキスト | 現在の会話・ファイル操作ログ・直近の意思決定 | セッション終了で消滅 |
| 永続ファクト | ユーザー設定・ルール・累積知識・好み | 永続 |
| 手続きスキル | 再利用可能な手順・ツール定義・最適化済みプロンプト | 永続・進化可能 |

**Hermes Agent の実績**
- スキル自己生成により同種タスクを **40% 高速化**（人手介入なし）
- 6プラットフォーム対応・40以上のツール統合

## 4. Human-in-the-loop 設計

### Pepabo 方式のヒューマンチェックポイント

```
AI（夜間処理）
  → SQLite（書き込み専用）
  → カンバンUI（読み取り専用）
  → 人間が朝に確認・優先度変更
  → 狭いAPIで状態遷移のみ操作可
```

- AI と人間の役割を明確に分離
- 人間の操作は「状態変更」に限定（AI の処理内容には干渉しない）

### Hermes Agent のレビューゲート

| 操作 | 自動化 | 人間レビュー |
|------|--------|--------------|
| スキル生成（新規） | ✅ 完全自動 | 不要 |
| スキル進化（既存更新） | ❌ | ✅ 必須 |
| タスク実行 | ✅ | 不要（結果確認のみ） |

## 5. タスク管理・優先順位付け

### Obsidian Kanban との統合

- Markdown ベースのカンバンボードで AI 生成タスクを可視化
- フロントマター `status: Todo / In Progress / Done` で状態管理
- シェルコマンド経由で Kiro（AI コーディングエージェント）へ直接タスク送信
- `[[wikiリンク]]` でノート間をナビゲート

### タスクキューの設計原則

1. **一方向データフロー**: 人間がタスクを投入 → AI が処理 → UI で確認
2. **バックプレッシャー制御**: 過剰タスク投入を防ぐ待機状態の可視化
3. **優先度の明示化**: 緊急度・重要度をバッジやカラーで表示
4. **失敗の可視化**: AI が処理できなかったタスクを明示し人間が介入
