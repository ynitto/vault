# AIの成果物に責任を取る方法

## 概要

AIコーディング全盛期において、エンジニアは「自ら書く者」から「AIをマネジメントする意思決定者」へ役割が変化しています。AIに全責任を負わせるのではなく、人間が責任者として、プロンプト・環境整備・教育を通じた信頼構築を行う重要性を説いています。

## 詳細

- AIコーディング全盛期において、エンジニアは「自ら書く者」から「AIをマネジメントする意思決定者」へ役割が変化しています。AIに全責任を負わせるのではなく、人間が責任者として、プロンプト・環境整備・教育を通じた信頼構築を行う重要性を説いています。
- 要点
- **AIのマネジメント**: AIを新人部下のように扱い、タスクを切り出し、意思決定レベルで品質と方向性を管理する。
- **環境と教育の重要性**: プロンプトでの前提提示や、Linter・コンテナ等の隔離された検証環境（ガードレール）の構築がAIへの信頼の基盤となる。
- **コードレビューの再定義**: 品質担保ではなく「AIと人間の認識のズレ」を確認し、指示や環境をチューニングするプロセスと捉える。
- **GitHub Copilotの活用**: カスタム指示、エージェント機能、DevContainer等を活用してマネジメントを実装する。
- **若手エンジニアの課題**: 実装経験が積みにくくなる分、理論学習と探求心で補い、ブラックボックス化を避ける必要がある。
- **最終的な責任**: AIの失敗はマネージャーである人間の責任。誠実な対応と改善が信頼を築く。
