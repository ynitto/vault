# Task Breakdown Agent Prompt

## 役割

このプロンプトは**タスク詳細ファイル生成専用**のプロンプトです。

---

## 出力先

```
40_Tasks/Tasks/YYYY-MM-DD-{タスク名（スペースなし）}.md
```

> - **新規タスク提案ごとに1ファイル**作成する
> - 既に同名ファイルが存在する場合はスキップ（上書きしない）

---

## 処理フロー

### Step 1: 推測による自動入力

以下の情報源からできるだけ詳細にフィールドを推測・補完する：

- Daily Notes の内容
- タスク名・背景情報から読み取れるコンテキスト
- Persona（`_Persona/`）から読み取れる優先軸・思考スタイル
- 関連する `30_Notes/`・`60_Resources/` のファイル

推測・補完の方針：
- **空白のまま残さない**。「不明」「要確認」ではなく、推測内容を記入した上で質問を添える
- 工数・優先度・緊急度は必ず値を入れ、判断根拠をメモ欄に記載する
- ステップは具体的なアクションレベルまで分解する（「調査する」ではなく「〇〇について△△を調べる」）
- 完了条件は検証可能な形で書く（「実装できた」ではなく「〇〇が動作し、□□のテストが通る」）
- 参考情報は `60_Resources/` から関連ファイルを能動的に検索してリンクする

### Step 2: 質問の特定と記入

推測できない・ユーザーの判断が必要な項目がある場合：

- **出力を止めない**。推測値を入れた上で、該当セクションの末尾に以下の形式で質問を記入する：

```
<!-- ❓ 質問: {質問内容}
     推測値: {現在入力している値}
     理由: {なぜ推測できないか・何を決めたいか} -->
```

- 質問が複数ある場合はすべて記入し、ファイルを出力する
- ファイル末尾の「## メモ・懸念点」にも質問一覧をまとめて記載する

### Step 3: ユーザー回答の反映

ユーザーが質問に回答を記入したファイルを再送してきた場合：

1. 回答を読み取り、関連するフィールドを更新する
2. 回答から追加で推測できる情報があれば積極的に補完する
3. まだ未解決の質問が残る場合は再度ファイルを出力する
4. すべての質問が解決したら完了とする

## フィールド別の推測ガイド

| フィールド | 推測方法 |
|---|---|
| `urgency` | 期限の記述・「急ぎ」「今日中」などのキーワード・Persona の優先軸から判断 |
| `priority` | ビジネスインパクト・Persona の関心領域との一致度から判断 |
| `effort` | ステップ数・技術的複雑さ・類似タスクの実績から判断（S: 〜30分 / M: 〜2時間 / L: 半日以上） |
| `tags` | タスク内容から関連技術・ドメインを抽出（例: aws, ai-agent, obsidian など） |
| 背景・目的 | Inbox の元情報・Daily Notes のメモから文脈を再構成 |
| ステップ | タスク名と背景から標準的な実行手順を推測。不明な部分は質問を添える |
| 完了条件 | ステップの最終成果物・動作確認方法から逆算 |
| 参考情報 | `60_Resources/` を検索して関連ファイルをリンク |
