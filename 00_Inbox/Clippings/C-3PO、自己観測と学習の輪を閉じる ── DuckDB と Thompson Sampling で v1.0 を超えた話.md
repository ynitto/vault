### 記事の要約
本記事は、自作のClaude Code用ツール「C3 (Claude Code Conductor)」において、「自己観測・学習・行動変更」のフィードバックループを実装し、v1.0からv1.5.2までのリリースを通じて機能を成熟させた経緯を解説しています。

### 主な要点
- **自己改善ループの構築**:
  - **観測**: サブエージェントの動作ログやイベントを記録。
  - **集約**: SQLiteとDuckDBのハイブリッド構成により、複雑な分析を可能にするストアを構築。
  - **学習**: Thompson Samplingを用いたTier（AIモデル）選定の最適化。
  - **行動変更**: 学習結果を基にモデルの動的切替やタスクルーティングの自動化を実現。

- **技術的な工夫と教訓**:
  - **SQLによる安全性**: 状態遷移の保護や終端制御をアプリケーション層ではなくSQLレイヤで完結させることで堅牢性を確保。
  - **運用の自動化**: `c3 status` CLIの導入や、非TTY環境でのサマリ出力による監視の効率化。
  - **設計の教訓**: 段階的リリースとdry-runの重要性、Skillツールにおける引数伝達の制約（envではなくargsを利用）、ドキュメントと実装の乖離への注意など。

- **成果**:
  - 合計10個の主要機能（F-001〜F-010）が揃い、エージェントが過去の成功・失敗から「学習し、行動を変える」サイクルが完成しました。
