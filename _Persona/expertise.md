# Expertise

最終更新: 2026-05-05

## 主要関心領域

### AI エージェント・LLM 活用
- Kiro CLI を AI エージェント基盤として活用することへの関心が継続している
- 仕様優先アプローチへの親和性が高い（Kiro の設計思想と一致）
- Claude Code / Agent Skills / Copilot 周辺の運用設計と品質管理
- LLM の長期記憶・第二の脳・Knowledge Base 的な構造化記憶
- kiro-loop の設計・自律実行基盤の構築（継続的な深化テーマ）
- Opus 4.7 など新モデルの特性に合わせたプロンプト設計への関心が高い
- コスト最適化（モデルルーティング: Haiku/Sonnet → Opus）を運用設計の一部として意識している
- RAG の代替手法（階層型RAG / Corpus2Skill）への関心。既存の仕組みへの応用可能性を常に探る

### Obsidian・PKM
- Obsidian を AI エージェントの作業基盤にすること
- AI エージェント運用を Markdown / Git / Obsidian で構造化する志向が強い
- 知識は再利用単位に圧縮し、タスクは詳細ファイル化して保持する好みがある

### AWS・インフラ
- CloudFront mTLS / KVS のような再利用性の高いインフラ設計パターンへの関心
- ECS / Lambda / CloudFormation 等の AWS サービス全般

### 開発プロセス
- 新機能導入時も、まず運用ルール・評価軸・保全先を整える傾向がある
- CI/CD・GitLab・Jenkins 等の自動化基盤

## 技術スタック（推定）
- 言語: Python, TypeScript/JavaScript, Node.js
- インフラ: AWS (ECS, Lambda, CloudFront, CloudFormation)
- ツール: Kiro CLI, Claude Code, Obsidian, Git/GitLab
