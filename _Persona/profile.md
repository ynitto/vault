# Profile

最終更新: 2026-05-05

## コミュニケーションスタイル

- 日本語で回答することを好む
- 簡潔・構造化された出力を好む（箇条書き・表・見出しを適切に使う）
- 抽象論より具体的なアクション・コマンド・ファイルパスを好む

## 行動・判断パターン

- 情報収集だけで終わらせず、すぐ実行可能な運用フローへ落とすことを重視する
- 新機能やツールそのものより、「壊れにくい運用」「継続改善の仕組み」に強い関心がある
- 同系統の記事が多い場合、単発知識より統合ノート化して再利用可能な形に圧縮する志向がある
- 「まず枠組みを決めてから増やす」スタイルが強い
- 読んだ情報を即タスク化・ノート追記する「読んだら即アクション化」の傾向がある
- 抽象的な課題を構造化して保持するスタイル（例: kiro-loopの設計課題を3系統に整理）
- 完璧主義ではなく「できるときにやる」スタイル。キャリーオーバーを許容する
- 既存ノートへの追記を新規ファイル作成より優先する
- GW・休日でも情報整理の習慣を維持している（朝の時間帯に実行することが多い）

## タスク管理スタイル

- タスクは Daily → 詳細ファイル → Kanban のように段階化された形が相性が良い
- S/M/L の工数評価を自然に使いこなしており、タスクの重さを直感的に把握している
- 思考の整理では raw source / notes / tasks の層分けを好む

## 情報整理スタイル

- daily_analysis → knowledge_builder の2段階情報処理フローが定着している
- 一次情報だけでなくコミュニティの反応（Reddit等）も含めて収集する傾向がある
- 情報収集サイクルは不定期だが、週1〜2回程度のペースで実行している
