# 改善案・拡張要件リスト

> 参考実装: [ynitto/sandbox](https://github.com/ynitto/sandbox)（現在の自動化・エージェント実験リポジトリ）  
> 参考記事: [[AI活用事例まとめ]]

> [!note] 注記
> ynitto/sandbox リポジトリへのアクセスが制限されているため、vault の既存実装（テンプレート・Kiro連携・Obsidianワークフロー）と参考記事の知見をベースに要件を定義しています。sandbox の実装詳細に合わせて適宜調整してください。

---

## 現状の整理

### 現在実装されているもの（vault 観察より）

- Obsidian + Markdown によるナレッジ管理
- `kanban_task.md` テンプレート（Kiro への送信ボタン付き）
- `daily_lookback.md` テンプレート（日次振り返り）
- `Web/` フォルダへの記事ブックマーク保存
- Kiro（AI コーディングエージェント）との Shell コマンド連携

### 課題・ギャップ

| 課題 | 影響 | 優先度 |
|------|------|--------|
| AI タスク処理が手動起動 | 夜間の自動処理ができない | 高 |
| タスクキューが存在しない | 複数タスクの管理・可視化が困難 | 高 |
| エージェントの実行結果が散逸 | 成果の蓄積・再利用ができない | 高 |
| スキル・手順の記憶機能がない | 同じタスクを毎回ゼロから処理 | 中 |
| 記事情報がリンクのみ | 内容の検索・分析・活用ができない | 中 |
| 日次振り返りが手動 | 継続性が低い | 低 |

## 改善案・拡張要件

### REQ-01: 夜間自動化パイプラインの構築

**概要**: Claude Code / Kiro を夜間に自動起動し、タスクキューを消化するパイプラインを実装する

**要件**
- [ ] cron または Claude Code Routines によるスケジュール実行
- [ ] タスクキュー（SQLite）への自動タスク投入（GitHub Issues, Obsidian タスクなど）
- [ ] 夜間実行のログを翌朝確認できる形で出力
- [ ] 実行結果をカンバンボードに反映

**参考**: [[2026-04-18-夜間自動化基盤]]

---

### REQ-02: SQLite タスクキューの実装

**概要**: タスクの投入・管理・追跡を行う軽量なデータストアを構築する

**要件**
- [ ] タスク CRUD（作成・読取・更新・削除）の API
- [ ] タスクスキーマ: `id, title, body, status, priority, source, created_at, updated_at`
- [ ] ステータス管理: `todo → in_progress → done / failed / waiting`
- [ ] 書き込みは承認済みソース（cron, Obsidian）のみ
- [ ] 人間による状態変更は専用エンドポイント経由のみ

**参考**: [[2026-04-18-SQLiteタスクキュー]]

---

### REQ-03: 軽量カンバン Web UI

**概要**: SQLite のタスクキューを可視化する Python http.server ベースのダッシュボード

**要件**
- [ ] SQLite からの読み取り専用表示
- [ ] ステータス列によるカンバンビュー
- [ ] ソース別バッジ（GitHub Issue, Obsidian, cron）
- [ ] 優先度カラーリング
- [ ] 人間による状態変更ボタン（待機・優先度変更）
- [ ] 自動リフレッシュ（ポーリング or SSE）

**参考**: [[2026-04-18-カンバンWebUI]]

---

### REQ-04: スキル記憶システムの設計

**概要**: Hermes Agent の手続きスキルを参考に、AI が習得した手順を再利用可能な形で永続化する

**要件**
- [ ] スキルファイルフォーマットの定義（Markdown / YAML）
- [ ] スキルの種類: コマンド手順・ツール設定・プロンプトテンプレート
- [ ] Obsidian vault 内の `Skills/` フォルダでスキルを管理
- [ ] スキルファイルに `[[wikiリンク]]` で関連タスク・事例を紐付け
- [ ] スキル検索・呼び出し CLI / API

**参考**: [[2026-04-18-スキル記憶システム]]

---

### REQ-05: 自己評価チェックポイントの実装

**概要**: AI エージェント実行後に自己評価を行い、スキル保存の判断と品質改善を自動化する

**要件**
- [ ] N ツールコールごとに評価チェックポイントを挿入
- [ ] 評価観点: 効率性・再利用性・エラー率・実行ステップ数
- [ ] 「保存価値あり」判定時にスキルファイルを自動生成
- [ ] 評価結果を SQLite に記録（トレースログ）
- [ ] 評価サマリーをカンバン UI に表示

**参考**: [[2026-04-18-自己評価チェックポイント]]

---

### REQ-06: Obsidian ↔ Kiro 連携の強化

**概要**: 既存の kanban_task テンプレートを拡張し、Kiro への自動タスク送信・結果受信を強化する

**要件**
- [ ] タスク作成時に自動でタグ・優先度を推定（LLM 分類）
- [ ] Kiro の実行結果を Obsidian ノートに自動書き戻し
- [ ] `受け入れ条件` チェックボックスの自動検証
- [ ] 完了タスクのアーカイブ（`Archive/` フォルダへ移動）
- [ ] タスク完了時に関連スキルファイルへの `[[リンク]]` 自動追加

**参考**: [[2026-04-18-Obsidian-Kiro連携]]

---

### REQ-07: GitHub Actions × Claude Code Action の設定

**概要**: GitHub Issues をトリガーに Claude Code が自動でコード修正・PR 作成を行うワークフローを構築する

**要件**
- [ ] `claude` ラベル付き Issue で Claude Code Action を自動起動
- [ ] 夜間バッチ Issue の自動作成（定期的なリファクタリング・テスト追加など）
- [ ] 作成された PR に実行サマリーをコメント
- [ ] PR 作成後に Obsidian カンバンへタスクを自動投入
- [ ] 実行コストの記録・アラート

**参考**: [[2026-04-18-GitHub-Actions設定]]

---

### REQ-08: スキル進化パイプライン

**概要**: 既存スキルファイルを実行トレースから継続的に改善する GEPA インスパイアのパイプライン

**要件**
- [ ] 実行トレースから失敗・非効率パターンを抽出
- [ ] LLM によるスキル改善案の生成（複数バリアント）
- [ ] バリアント評価（実行速度・成功率）
- [ ] 改善版スキルの PR 作成（人間レビュー必須）
- [ ] スキルのバージョン管理（Git）

**参考**: [[2024-04-18-スキル進化パイプライン]]

---

### REQ-09: Web 記事の自動要約・ノート化

**概要**: ブックマークした記事を自動でフェッチし、Obsidian ノートとして構造化する

**要件**
- [ ] `bookmarks.json` の変更を検知して記事を自動取得
- [ ] LLM で要約・カテゴリ分類・タグ付けを自動実行
- [ ] `Web/` フォルダに構造化 Markdown として保存
- [ ] AI活用事例まとめへの自動リンク追加
- [ ] 重複記事の検出・マージ
