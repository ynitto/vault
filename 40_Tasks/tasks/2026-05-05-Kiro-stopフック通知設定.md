# Kiro stopフック通知設定

## 背景・目的
Kiro CLIのエージェント処理完了時に通知音を鳴らすことで、長時間タスク実行中の完了見逃しを防ぐ。`stopフック` を使うことで、LLMの判断に依存せず確実に通知できる。macOS環境では `afplay` コマンドで実装可能。

## 条件
### 前提条件
- macOS環境（afplayコマンドが利用可能）
- Kiro CLIの設定ファイル（`.json`）へのアクセス権限

### 制約条件
- 全エージェント共通で管理できる構成にする（個別設定の重複を避ける）
- 通知音は `/System/Library/Sounds/` の既存音源を使用（追加インストール不要）

## やること（ステップ）

- [ ] ステップ1: `~/.kiro/` 以下の設定ファイル構造を確認する
- [ ] ステップ2: 通知音再生スクリプト `notify.sh` を作成する（`afplay /System/Library/Sounds/Glass.aiff`）
- [ ] ステップ3: Kiro の設定ファイルの `hooks.stop` フィールドにスクリプトを指定する
- [ ] ステップ4: テスト実行して通知音が鳴ることを確認する
