# Opus 4.7プロンプト最適化スキル設計

## 背景・目的
Claude Opus 4.7 はモデル特性の変化（adaptive thinking一本化、トークナイザー変更）により、従来の指示の型では最大性能を引き出せない。「5スロットブリーフ」（Goal / Constraints / Acceptance criteria / Context / Validation）を自動生成するスキルを設計・実装することで、Kiro CLI上でのOpus 4.7活用を最適化する。

## 条件
### 前提条件
- Kiro CLI のスキル開発環境が整っていること
- Claude Opus 4.7 の API アクセスが可能であること
- `skill-creator` スキルを活用して実装する

### 制約条件
- 既存スキルとの重複を避ける（`empirical-prompt-tuning` との役割分担を明確にする）
- スキルは `generate`・`audit`・`route` の3モードを持つ設計とする

## やること（ステップ）

- [ ] ステップ1: 参考記事（Opus 4.7向けAgent Skill）の実装詳細を精読し、5スロットブリーフの仕様を整理する
- [ ] ステップ2: `empirical-prompt-tuning` スキルとの役割分担を確認し、重複しない設計方針を決める
- [ ] ステップ3: `skill-creator` を使ってスキルの SKILL.md ドラフトを作成する
- [ ] ステップ4: `generate` モード（数行の依頼 → 80行ブリーフ変換）を実装する
- [ ] ステップ5: `audit` モード（既存プロンプトの品質チェック）を実装する
- [ ] ステップ6: `route` モード（Haiku/Sonnet → Opus へのエスカレーション制御）を実装する
- [ ] ステップ7: 実際のタスクで動作確認し、品質向上を検証する
