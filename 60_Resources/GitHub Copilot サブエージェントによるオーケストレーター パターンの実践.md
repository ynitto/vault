### 概要
VS Code v1.107の新機能である「カスタムエージェント（.agent.md）を`runSubagent`で直接呼び出す機能」を活用し、開発ワークフロー全体（Issue作成〜PR生成）を自動化するオーケストレーター構築の実践解説。

### 要点まとめ
- **サブエージェント化のメリット**
  - コンテキスト肥大化の防止、責任の明確化、保守性の向上。
  - レビュープロセス等の特定フェーズの独立による品質確保。

- **設計のポイント：責任分離**
  - 各エージェントに「Issue」「Plan」「Impl」「Review」「PR」の役割を特化させ、オーケストレーターは指揮者に徹する。
  - エージェント定義には必要最小限のツールのみを指定（Context Rot対策）。

- **開発ワークフローの自動化**
  - オーケストレーターが各サブエージェントを順次呼び出すことで、一貫した開発プロセスを実現。
  - `gh`コマンドを積極的に採用し、MCPツール利用に伴うコンテキスト消費を抑制。

- **運用上の工夫**
  - ドキュメント（CONTRIBUTING.md等）の外部化と参照により、エージェント定義をシンプルに保つ。
  - 「ユーザーの意図を自ら解釈させない」等の制約プロンプトにより、意図しない挙動を抑止。

- **今後の課題**
  - 本記事はハッピーパスを前提としており、エラー発生時のリカバリーや人間による承認判断プロセス（エスケープハッチ）の実装が実運用の鍵となる。
