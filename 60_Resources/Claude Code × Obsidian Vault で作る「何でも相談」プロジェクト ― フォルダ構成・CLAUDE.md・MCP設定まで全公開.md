### 概要
Claude CodeとObsidianを連携させ、プロジェクト成果物を自動整理・蓄積する「何でも相談-pj」の構築手法。プロジェクトルートを汚さず、知識をグラフ化・管理する効率的な運用術を解説しています。

### 要点まとめ

#### 1. 設計の3原則
* **Vault集約**: 成果物はすべてObsidian Vault配下に保存。
* **ルール自動化**: `CLAUDE.md`に階層ルールを明記し、Claude自身に配置管理をさせる。
* **ポータブル設計**: OS間移行を容易にするため、あえてGitを使わずフォルダ丸ごと管理。

#### 2. プロジェクト構造の肝
* **`CLAUDE.md`の活用**: 新規ファイルの置き場を表形式で定義。これにより「どこに保存するか」の指示が不要に。
* **`obsidian-vault/`の役割**: 相談、コーディングメモ、技術調査、アーカイブをフォルダで明確に分類。

#### 3. 設定による最適化
* **`settings.json`**: モデルの固定、推論深度（effortLevel: xhigh）の設定、不要な思考調整の無効化。
* **`settings.local.json`**: 各端末ごとの許可リスト（Web検索等）を分離し、プロンプト表示を抑制。

#### 4. 運用ルールのポイント
* **MCP制限**: コンテキスト節約のため、プロジェクト専用のMCP（AWS Docsなど）に絞る。
* **OS間移行ルール**: 絶対パスを避け、パス区切りを`/`に統一することでMac/Windows双方に対応させる。
