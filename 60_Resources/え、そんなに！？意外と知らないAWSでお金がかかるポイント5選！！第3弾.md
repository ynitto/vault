### 概要
AWSの各サービスを利用する際に、意図せず高額なコストが発生してしまうリスクと、その対策方法を理論値ベースで解説した記事です。

### 注意点
- 提示金額は検証時点（2020年2月）の東京リージョンの理論値。
- サービスそのものが悪いわけではなく、設定や設計によるコスト管理の重要性を説くもの。

### コスト管理が必要な主要サービスと対策

#### 1. AWS Lambda
- **リスク**: 無限ループ等による意図しない大量実行・同時実行。
- **対策**: `Invocations`メトリクスの監視、異常時のアラート設定、同時実行数の制限、Saving Plansの活用。

#### 2. Amazon CloudWatch（Alarm）
- **リスク**: 無料枠を超えた設定や高解像度（10秒）メトリクス、異常検出機能の利用によるコスト増。
- **対策**: 不要なアラームの削除、閾値の再検討、AutoScaling設定の最適化。

#### 3. Amazon CloudWatch（カスタムメトリクス）
- **リスク**: 高頻度でのAPI呼び出し（PutMetricData）によるリクエスト課金。
- **対策**: 不要なメトリクスの整理、APIリクエスト頻度の削減。

#### 4. AWS PrivateLink
- **リスク**: エンドポイント（ENI）ごとの時間単位の課金と通信料。
- **対策**: VPC内部通信が必須でない場合はNAT Gateway経由の通信も検討。

#### 5. Amazon EBS
- **リスク**: 未アタッチの状態でも維持費が発生。
- **対策**: `DeleteOnTermination`（終了時に削除）を有効化し、長期間不要な場合はスナップショット化して削除。
