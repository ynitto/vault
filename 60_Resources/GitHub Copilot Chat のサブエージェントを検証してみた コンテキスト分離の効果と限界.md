### GitHub Copilot Chat サブエージェントの概要と活用
GitHub Copilot Chat のサブエージェント（`agent/runSubagent`）は、複雑なタスクを独立したセッションで処理するための機能です。

#### 主なメリット
- **コンテキスト分離**: メインのチャットセッションを肥大化させず、特定のタスクに集中したコンテキストを維持できる。
- **並列処理**: 複数のサブエージェントを並列実行可能（要 VS Code Insiders）。
- **効率化**: 複雑な調査、コード検索、段階的なタスクにおいて、メインエージェントの負荷を軽減できる。

#### 活用におけるポイント
- **効果的な場面**:
  - レビューや調査など、独立した視点が必要なタスク。
  - ファイル単位で独立して処理可能な作業。
  - 計画・実装・レビューなどのフェーズが明確に分かれるワークフロー。
- **注意点**:
  - **起動コスト**: サブエージェントの生成には固定のオーバーヘッドがあり、単純なタスクでは逆に効率が悪化する場合がある。
  - **総コスト**: メインセッションの消費量は減るが、サブエージェント側の処理分を含めると総トークン消費量は増加傾向にある。
  - **情報共有の制約**: サブエージェントはステートレスであり、最終結果以外の途中経過はメインエージェントから参照できない。

#### 実践的なアドバイス
- プロンプトで `#tool:agent/runSubagent` を明示し、具体的な `prompt`（詳細なタスク記述）と `description` を定義することで精度が向上します。
- カスタムエージェントを呼び出す場合は、設定で `\"chat.customAgentInSubagent.enabled\": true` を有効にする必要があります。
