## 概要
本書は、AWS の各サービスにおける「ベストプラクティスアラーム」の設定推奨事項をまとめた技術ガイドです。

## 要点

### 1. アラーム設定の目的
- サービスごとのメトリクスを活用し、異常発生を早期に検知する。
- 可用性やパフォーマンスの低下を未然に防ぎ、障害発生時の迅速なトラブルシューティングを支援する。

### 2. 主要な構成要素
推奨される各アラーム設定には、以下の情報が含まれています。
- **ディメンション**: モニタリング対象の特定。
- **目的**: 設定の意図と監視すべき状況。
- **統計・しきい値**: 警告を発する基準値。
- **期間・評価期間**: 監視のタイミングと精度。

### 3. 主な対象サービス
API Gateway, EC2, ECS, RDS, S3, SQS, Lambda, ElastiCache, Kinesis など、主要な AWS マネージドサービスを網羅。

### 4. 運用のヒント
- **しきい値の調整**: 履歴データや各環境のワークロード特性（通常時のトラフィック量など）に基づいて、しきい値を調整することが推奨される。
- **過剰検知の防止**: 感度が高すぎる設定は不要なアラートを増やすため、許容可能なエラー率やレイテンシーを定義する必要がある。
- **トラブルシューティング**: アラーム発生時は、各セクションで提供されるドキュメントやトラブルシューティングガイドを活用する。
