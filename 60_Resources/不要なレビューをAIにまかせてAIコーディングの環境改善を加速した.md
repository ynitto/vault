### 概要
Findy Team+の開発において、AIコーディングの普及に伴い増大した「機械的なコード修正」のレビュー負荷を軽減するため、AIを活用したプルリクエスト（PR）自動承認システムを構築した話。

### 要点
- **背景と課題**
  - AI活用で開発スピードが向上したが、付随する大量のPR（リファクタリングや型定義変更など）のレビューが人間にとって過度な負担となっていた。
- **解決策：AI自動レビュー**
  - 「レビュー価値がないPR（儀式的な確認）」を特定する分類器を作成。
  - Kent Beckの著書『Tidy First?』の概念を参考に、振る舞いを変えない構造的リファクタリング（リネーム、移動、整理など）を自動判定の対象とした。
- **実装と工夫**
  - **技術スタック**: GitHub Actions と Claude Code Actions を使用。
  - **戦略**: 大規模なPRはサンプリング（代表例の確認）を行うことでコストと負荷を最適化。
  - **運用**: 1件あたりの判定コストは$0.2〜$0.5程度。まずはフロントエンドチーム限定で導入。
- **成果**
  - レビュープロセスを自動化することで、作成者が単独でマージ可能な範囲を拡大し、開発者体験とリファクタリングのモチベーションを向上させた。
