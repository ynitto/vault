### 要約
本記事は、プロンプトの改善効果を可視化・検証できるClaude Codeプラグイン「Rashomon」の紹介です。プロンプトエンジニアリングの試行錯誤を自動化し、改善が及ぼす影響を直感的に理解することを目指しています。

### Rashomonの主な特徴
- **自動分析・最適化**: よくあるプロンプトの問題パターンを検出し、ベストプラクティスに基づいて最適化。
- **並列検証**: Git Worktreeを活用し、オリジナルと最適化後のプロンプトを隔離環境で同時実行。
- **結果レポート**: 両方の実行結果を比較し、所要時間やコード変更の有無を明確化。
- **知見の蓄積**: 比較結果からフィードバックを学び、プロジェクト固有のナレッジとして活用。

### 導入メリット
- 「追加して」といった指示による過剰な実装を抑制できる。
- 「必要なければ変更しない」という選択肢をLLMに与え、効率的なタスク遂行が可能になる。
- プロンプトの書き方が実行結果にどう影響するかを具体的・客観的に把握できる。

### 検出する8つの問題パターン
- 負の指示の回避、曖昧な指示の明確化、出力形式の指定、構造化、コンテキスト補完、タスク分割、バイアス回避、不確実性の許容など。
