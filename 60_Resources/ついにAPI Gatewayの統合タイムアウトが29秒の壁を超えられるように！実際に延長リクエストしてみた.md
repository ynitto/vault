### API Gateway 統合タイムアウト延長の概要
AWS API Gateway（REST/Regional/Private API）において、これまで一律29秒だった統合リクエストのタイムアウトが、Service Quotasを通じて延長可能になりました。

### 延長の申請手順
- **方法:** AWS Service Quotasより「timeout」で検索し、リージョンごとにアカウントレベルで申請。
- **注意:** 自動承認ではない場合、詳細なユースケース（RPS、予想レイテンシ、ペイロードサイズ等）の提出を求められるため、事前に準備が必要です。
- **所要時間:** サポートとのやり取りを含め、承認まで約1週間程度の期間を要しました。

### 実装のポイント
- **検証:** Lambdaのタイムアウト設定自体も忘れずに延長する必要があります。
- **注意喚起:** 同期的な長時間処理はシステム負荷やレスポンス待ちの観点から非推奨。可能な限り非同期処理（Step Functionsの利用など）での設計を推奨します。
