### 記事要約
GitリポジトリとClaude Codeを組み合わせ、自身の思考やプロジェクト管理を「第二の脳（Second Brain）」として構築・自動運用する方法を解説した実践録です。Notion等の既存ツールを超え、AIが自律的に整理・判断・実行を行う仕組みを構築しています。

### 要点まとめ

#### 1. なぜGitリポジトリなのか
- **AIとの相性**: Claude Codeによるファイル直接操作が可能。
- **拡張性**: フォルダ構造、ワークフローを自由に定義可能。
- **統合管理**: GitHub IssuesやActionsとシームレスに連携。

#### 2. 設計思想と構成
- **brain（思考）とproject（実装）の分離**: 目的（Why/What）と実装（How）を分け、単一の事実源を確保。
- **CLAUDE.md**: AIの役割を「思考パートナー」に定義し、自律的な判断・構造化を指示。
- **階層化フォルダ**: Inboxからアーカイブまで、情報の「流れ」を番号付きフォルダで管理。

#### 3. 自動化と自律化の仕組み
- **14コマンド × 13エージェント**: 運用をコマンド化し、専門エージェントに思考を委譲。
- **PM Layer**: 「プロジェクト管理の自動介入」を導入し、停滞パターンの診断から次アクションの提案までを自動化。
- **3層自律アーキテクチャ**:
  - Layer 1: コマンド実行
  - Layer 2: オーケストレーターによる判断・優先度付け
  - Layer 3: GitHub Actionsによる常時稼働（構想）

#### 4. 運用上の教訓
- **ガードレール**: AIの自律性は段階的に権限を上げ、ガードレール（安全ルール）で制限する。
- **一元化**: 最初に「Issuesはbrainに集約する」と決めることが最も重要。
- **役割**: AIを全知全能のツールにするのではなく、適切な情報へアクセスし、意思決定を補助させる「コンテキストエンジニアリング」として活用する。
